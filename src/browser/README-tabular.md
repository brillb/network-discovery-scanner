# Network Scanner Dashboard Browser

This component provides a premium, responsive local web interface to visualize the configured discovery database and configuration evidence plain-text files generated by the scanner.

## Overview

As defined in the core [`architecture.md`](../../docs/architecture.md), the scanner orchestrator produces two main outputs over runtime:
1. A database defined by `db.yaml` capturing host state, inventory, interfaces, and CDP/LLDP neighbor maps.
2. `discovered_device_evidence_YYYYMMDD_HHMMSS/` : Folders containing raw running-config and command dumps.

Because the system allows multiple scans over time, a single device IP might map to multiple temporal snapshots. The `tabular_browser.py` solves for exploring this data organically without needing raw SQL queries.

## Features

- **Config-Driven Database Loading**: Point the browser at `db.yaml` and it will open the configured SQLite, MySQL, MariaDB, or PostgreSQL scanner database.
- **Premium Glassmorphic UI**: High-end aesthetic with dark mode and native CSS data tables.
- **Client-Side Sorting**: Instantly sort devices by IP, Type, Hostname, or Model without server roundtrips.
- **Credential Visibility**: Device detail views surface the latest stored SNMP and SSH credential references plus the current TCP/22 reachability state.
- **Time-Series Analysis**: When viewing a single device, select from a dropdown of previous scans to instantly diff the hardware states, neighbor changes, and compare running configurations side-by-side using the saved SSH output files.

## Usage

Ensure you have Flask installed:
```bash
pip install flask
```

Run the server by pointing `--dbconfig` at `db.yaml`. Use `--directory` for the folder where the evidence directories reside. If you omit `--directory` while using SQLite, the browser defaults to the SQLite file's parent directory.

```bash
python tabular_browser.py --dbconfig /path/to/db.yaml --directory /path/to/scan/results
```

You can optionally specify a port:
```bash
python tabular_browser.py --dbconfig /path/to/db.yaml --directory . --port 8080
```

## Testing with Mock Data

A sample generator is provided to scaffold testing without hitting a live network.

1. Ensure you are in the `src/browser/` directory.
2. Run `python generate_sample_data.py`. This creates a DB, a matching `db.yaml`, and 20 mock evidence configs.
3. Start the UI: `python tabular_browser.py --dbconfig .\\db.yaml --directory .`
4. Visit `http://127.0.0.1:5000` to interact with the mock findings.
